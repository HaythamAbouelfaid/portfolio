## Overview

This project analyzes a **real-world SMS phishing (smishing) campaign impersonating the United States Postal Service (USPS)**.

A suspicious SMS message was received claiming that a package could not be delivered due to incomplete shipping information. The message attempted to lure the recipient into clicking a malicious link to update delivery details.

To investigate the message using a structured security workflow, I built **SmishGuard**, a Python-based CLI tool that automatically analyzes suspicious SMS messages and extracts **Indicators of Compromise (IOCs)**.

The goal of this project was to simulate how a **SOC analyst or incident responder** would investigate a phishing message and document the findings.

## Malicious Attachment Reference

The message referenced a suspicious document attachment that appeared to be related to delivery verification.

<img src="/images/projects/project-01/02-smishing-attachment.jpg" alt="USPS Smishing Attachment Screenshot" />

Attackers often include **fake shipping notices or invoices** disguised as PDF documents to increase credibility and pressure the recipient to open the file.

Opening such attachments on a normal system can lead to:

- Credential theft
- Malware execution
- Data exfiltration
- Device compromise

## Smishing Campaign Analysis

The SMS message contained several high-confidence phishing indicators:

- Unknown sender phone number  
- Brand impersonation (USPS)  
- Urgent delivery failure message  
- Call-to-action link  
- Attachment reference  

These characteristics match common **delivery-themed smishing campaigns** used to steal credentials, payment information, or distribute malicious files.

By analyzing the message using a structured process, the campaign was identified as **high risk**.

## SmishGuard CLI Tool

To automate the investigation process, I developed **SmishGuard**, a Python command-line tool that triages suspicious SMS messages.

The tool extracts:

- Phone numbers  
- URLs  
- Domains  
- Attachment references  

It also assigns a **risk score** based on known phishing indicators.

Example command:

    python -m smishguard.cli --file samples/usps_smish.txt --from "+1 (705) 854-1876" --attachment "USPS-Notice-USD497.pdf"

Example output:

    Risk: HIGH
    Score: 100/100

The message was flagged due to multiple indicators including brand impersonation, urgent delivery language, malicious links, and attachment references.

## Safe Attachment Handling

Suspicious attachments should never be opened on a host system.

If analysis is required, the file should be transferred to an isolated environment such as a virtual machine or sandbox.

Example PowerShell command used to compute a SHA256 file hash:

    Get-FileHash .\USPS-Notice-USD497.pdf -Algorithm SHA256

The hash can then be submitted to threat intelligence platforms such as VirusTotal for additional analysis.

## Detection Engineering

A **Sigma detection rule** was created to identify similar delivery-themed smishing campaigns using common phishing keywords.

Example indicators used in the rule include:

- "Urgent Notice"
- "Unable to deliver"
- "USPS"
- "Redelivery"
- "Click here"

This rule can be adapted for:

- SIEM detection queries
- SMS or email security filters
- Threat hunting workflows
